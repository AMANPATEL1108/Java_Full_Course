Docker Setup Guide for Spring Boot + MySQL
This guide explains how to containerize your Spring Boot application with MySQL using Docker. You'll learn what each file does, how to build, and how to run everything.

📁 Files Overview
1. Dockerfile
Defines how to build your Spring Boot app into a Docker image.

dockerfile
# Use Java runtime (Eclipse Temurin is a reliable OpenJDK distribution)
FROM eclipse-temurin:17-jdk

# Copy the built JAR file from your local machine into the image
ARG JAR_FILE=target/*.jar
COPY ${JAR_FILE} app.jar

# Command to run the app when container starts
ENTRYPOINT ["java","-jar","/app.jar"]
Note: You must run mvn package first to create the JAR file in the target/ folder. Docker will copy that JAR into the image.

2. application.properties (default, for local development)
Used when you run the app outside Docker (e.g., mvn spring-boot:run).
It connects to MySQL on localhost:3307.

properties
spring.datasource.url=jdbc:mysql://localhost:3307/docker_test
spring.datasource.username=root
spring.datasource.password=mysql
3. application-docker.properties (for Docker environment)
This file is activated by the docker Spring profile.
It uses the hostname mysql (the service name defined in docker-compose) and port 3306 (the internal container port).

properties
spring.datasource.url=jdbc:mysql://mysql:3306/mydb
spring.datasource.username=root
spring.datasource.password=mysql
4. docker-compose.yml
Defines two services: mysql (database) and app (your Spring Boot application).

yaml
services:
  mysql:
    image: mysql:8
    container_name: mysql_db
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: mysql
      MYSQL_DATABASE: mydb          # Database name created at startup
    ports:
      - "3307:3306"                 # Host port 3307 → container port 3306
    volumes:
      - mysql_data:/var/lib/mysql   # Persistent data storage

  app:
    build: .                        # Build using Dockerfile in current directory
    container_name: spring_app
    restart: always
    depends_on:
      - mysql
    ports:
      - "8080:8080"
    environment:
      SPRING_PROFILES_ACTIVE: docker   # Activates application-docker.properties
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/mydb
      SPRING_DATASOURCE_USERNAME: root
      SPRING_DATASOURCE_PASSWORD: mysql

volumes:
  mysql_data:
🔧 How to Build and Run
Prerequisites
Docker and Docker Compose installed

Your Spring Boot project compiles successfully with Maven

Step 1: Build the JAR file (required before Docker build)
bash
# Clean and package the application (skip tests if database not available locally)
mvn clean package -DskipTests
This creates target/*.jar (e.g., Cloud-Devops-03-0.0.1-SNAPSHOT.jar).

Step 2: Build and start all services with Docker Compose
bash
docker-compose up --build
--build forces a rebuild of the app image (picks up any code changes).

Docker will:

Pull the MySQL 8 image if not present.

Create a network so containers can talk to each other.

Start the MySQL container first (because depends_on).

Build your Spring Boot image and start it.

Step 3: Run in detached mode (background)
bash
docker-compose up -d --build
Step 4: Check logs
bash
# All logs
docker-compose logs -f

# Only app logs
docker-compose logs -f app

# Only mysql logs
docker-compose logs -f mysql
Step 5: Stop and remove containers
bash
# Stop containers but keep volumes (data persists)
docker-compose down

# Stop and delete volumes (database data will be lost)
docker-compose down -v
🧪 Testing Your Application
Once running, access your Spring Boot app at:
http://localhost:8080

To verify database connectivity, you can exec into the MySQL container:

bash
docker exec -it mysql_db mysql -uroot -pmysql
Then inside MySQL:

sql
USE mydb;
SHOW TABLES;
EXIT;
🔍 Important Concepts Explained
Why two application.properties files?
Local profile (default): uses localhost:3307 – works when you run the app directly on your machine.

Docker profile: uses mysql:3306 – works inside Docker because containers can reach each other using service names.

How does the app know which file to use?
In docker-compose.yml, we set the environment variable SPRING_PROFILES_ACTIVE: docker.

Spring Boot then loads application-docker.properties in addition to application.properties (properties in the profile override the default ones).

Why is the MySQL port mapped to 3307:3306?
3306 is MySQL’s default container port.

Mapping to 3307 on the host avoids conflict if you already have MySQL running locally on port 3306.

Inside the Docker network, the app still connects to mysql:3306 (the container port). The host mapping is only for accessing MySQL from your local machine (e.g., using DataGrip or localhost:3307).

What if I want to change the database name or password?
Update both:

MYSQL_DATABASE in the mysql service environment.
SPRING_DATASOURCE_URL (and username/password) in the app service environment.
The application-docker.properties file (or just rely on environment variables – they take precedence).
