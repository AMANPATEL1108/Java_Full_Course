What are Window Functions? (Plain English)

Window functions calculate something across a group of rows
‚û°Ô∏è without collapsing rows like GROUP BY does.

So you still see every row, but with extra insights added.

Think of it as:

‚ÄúLook around this row at related rows, and compute something.‚Äù

Core Syntax (This part matters)

  PARTITION BY column
  ORDER BY column

Breakdown:

function() ‚Üí what you calculate (ROW_NUMBER, RANK, etc.)

PARTITION BY ‚Üí how rows are grouped (like GROUP BY, but no collapsing)

ORDER BY ‚Üí how rows are ranked or ordered inside each group

1Ô∏è‚É£ ROW_NUMBER()
What it does:

Gives a unique number

Always increases by 1

No ties (even if values are equal)

SELECT employee_id, department, salary,
       ROW_NUMBER() OVER (
         PARTITION BY department
         ORDER BY salary DESC
       ) AS row_num
FROM employees;

Result (HR only):
salary	row_num
70000	1
70000	2
60000	3

‚úÖ Use when you need:

‚ÄúTop N rows per group‚Äù

Your Example Explained
SELECT employee_id, department, salary,
       RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS rank_in_dept
FROM employees;


This means:

Split employees by department

Order salaries highest first

Rank employees inside each department

Keep all rows
