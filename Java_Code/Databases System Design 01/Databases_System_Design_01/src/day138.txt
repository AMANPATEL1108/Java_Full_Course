1. Multiple JOINs (what you already wrote ‚Äì ‚úÖ correct)

Used when data is spread across more than two tables.

SELECT o.id, c.name, p.product_name
FROM orders o
JOIN customers c ON o.customer_id = c.id
JOIN products p ON o.product_id = p.id;


What‚Äôs happening

orders is the base table

Join customers to get customer info

Join products to get product info

Each JOIN adds more columns, not more filters

2. Self JOIN (table joins to itself)

Used when rows in a table relate to other rows in the same table.

SELECT e1.name AS employee, e2.name AS manager
FROM employees e1
JOIN employees e2
  ON e1.manager_id = e2.id;


Key idea

e1 = employee

e2 = manager

Aliases are mandatory or SQL gets confused

3. JOIN vs WHERE condition (important distinction)
JOIN ‚Üí defines relationships
WHERE ‚Üí defines filters
‚ùå Bad / old-style (don‚Äôt do this)
SELECT *
FROM orders o, customers c
WHERE o.customer_id = c.id;

‚úÖ Proper JOIN
SELECT *
FROM orders o
JOIN customers c
  ON o.customer_id = c.id;

JOIN + WHERE together (very common)
SELECT o.id, c.name
FROM orders o
JOIN customers c
  ON o.customer_id = c.id
WHERE c.country = 'USA';


Rule of thumb

ON ‚Üí how tables connect

WHERE ‚Üí which rows you want

4. Practice Examples (answers)
üß© Practice 1: Employee ‚Üí Manager

Requirement:
Show employee name and manager name.

SELECT
  e.name AS employee,
  m.name AS manager
FROM employees e
LEFT JOIN employees m
  ON e.manager_id = m.id;


Why LEFT JOIN?

Employees without managers (e.g. CEO) still appear

Manager column will be NULL

üß© Practice 2: Order ‚Üí Customer ‚Üí Product

Requirement:
Show order id, customer name, and product name.

SELECT
  o.id AS order_id,
  c.name AS customer,
  p.product_name
FROM orders o
JOIN customers c
  ON o.customer_id = c.id
JOIN products p
  ON o.product_id = p.id;
